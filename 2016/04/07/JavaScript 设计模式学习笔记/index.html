JavaScript设计模式学习笔记

工厂模式

工厂模式是抽象的创建具体对象的过程。解决了创建相似对象的问题，未解决对象识别的问题。

构造函数模式

构造函数用来创建特定类型的对象。创建自定义的构造函数是将他的实例表示为一种特定的类型。

上述

  • ①声明函数
  • ②与声明函数逻辑上等价。
  • ③函数定义转移到构造函数外部，此处sayName全局函数，sayName包含一个指向函数外部的指针。
    ①②中的每个方法都要在每个实例上重新创建一遍。即每个Person实例都包含一个不同的function实例。③解决了①②两个函数做同一件事情的问题。但若对象要定义很多方法，就要定义很多全局函数。那么自定义的引用类型就没有封装性而言了。可通过原型模式解决。

    原型模式

    原型模式创建的每个函数都有个prototype(原型)属性，这个属性是个指针，指向一个对象。原型对象可以让所有对象实例共 享它所包含的属性和方法。

    单例模式 Singleton

    单例模式用来划分命名空间，减少页面中全局变量的数量，避免多人开发时代码冲突。
单例模式命名空间 JavaScript Namespacing

命名空间是单例模式在js中的一个应用。命名空间将对象作为另一个对象的属性，因此深度为一个或多个图层。命名空间保证全局变量数量最低。

组合模式 Composite

组合使用构造函数模式和原型模式。
构造函数模式用于定义实例属性，原型模式用于定义方法和共享的属性

动态原型模式

使用动态原型模式时，不能使用对象字面量重写原型。在创建了实例的情况下重写原型，会切断现有实例于原型间的联系。

寄生构造函数模式

使用new操作符把包装函数叫做构造函数外，与工厂模式是相同的。此模式用于在特殊情况下创建构造函数。比如想创建具有额外方法的特殊数组。

稳妥构造函数模式 Durable

除了使用sayName()方法外，没有其他方法访问name的值了
